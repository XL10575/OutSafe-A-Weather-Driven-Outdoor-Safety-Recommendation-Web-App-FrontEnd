# 为什么会“无历史同日数据”

当界面提示「暂无历史同日数据」时，表示**过去 N 年同日**的天气没有成功拿到或解析出有效数据，百分位会退化为参考值 50%。常见原因如下。

---

## 1. 请求失败（最常见）

- 前端会对「过去 N 年」的每一天各发 **1 次** Open-Meteo Archive 请求（例如 5 年 = 5 次请求）。
- 任一请求若 **HTTP 非 2xx**（网络错误、超时、限流 429、服务 5xx 等），当前实现会**静默丢弃**该次结果，只保留成功的。
- 若 **5 次全部失败**，则「成功条数」为 0，就会变成「无历史同日数据」。

**可排查**：看结果卡片里的「请求了 X 年同日数据，成功 Y 条」。若 Y=0，多半是网络或 Archive API 不可达/限流。

---

## 2. Open-Meteo Archive 限流或不可用

- Archive 为免费接口，可能有**请求频率/并发限制**。
- 连续多次点「获取户外安全建议」或同时开多个页面，容易触发限流（如 429），导致多次请求失败 → 成功 0 条。

**建议**：稍等再试，或减少「对比历史年数」以降低单次请求数。

---

## 3. 区域或日期覆盖

- 理论上 Open-Meteo 历史数据覆盖全球、且历史较长（约 1959 年起），**多数**地点和「过去 N 年同日」都应能查到。
- 个别偏远、海洋或特殊区域，或某些日期存在缺测/未入库，可能返回空或异常，解析后也会被视为「无有效历史」。

---

## 4. 前端解析不到“当日”小时

- 即便接口返回 200，若某年的 **hourly** 里没有与「该年当日」匹配的时间点（例如时区或日期切分不一致），`aggregateDayMetrics` 会得到 `null`，该年就不会进入有效历史列表。
- 若**所有年份**都解析不到当日，有效历史条数就是 0 → 表现为「无历史同日数据」。

---

## 小结

| 现象 | 可能原因 |
|------|----------|
| 成功 0 条 | 网络问题、Archive 限流或服务异常、请求全部失败 |
| 成功 &lt; 请求年数 | 部分请求失败，或部分年份无数据/解析失败 |
| 有成功但仍提示无历史 | 成功返回里没有解析出「当日」小时（少见） |

界面已显示「请求了 X 年，成功 Y 条」：**Y=0** 时重点看网络与限流；**Y&gt;0** 但仍无历史，多半是解析或数据格式问题，可再结合控制台或后端日志排查。
